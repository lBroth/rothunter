what it catches

33 detectors built for cross-file defects.

Single-file rules can't see what they don't read. Below is a sample from each of the six hard-spot categories rothunter ships ON by default — reachability, concurrency, AST clustering, contract drift, barrel ambiguity, taint flow.

high race-condition

Read-modify-write across `await`

CFG-aware detector flags the classic if (cond) read; await; write shape on a shared resource. LLM verdict rules out mutex / single-flight / scoped state.

high shared-db-write

Same column, two services

Two repos writing the same Prisma / Drizzle / TypeORM column without coordination. Cross-workspace in monorepo mode — the only place the race is visible.

high api-race

Async fan-out on a write endpoint

≥ 2 caller files hit the same `PUT /api/x` endpoint without an `If-Match` / single-flight guard. Catches the "last write wins" silent overwrite.

high unsanitized-input-to-sink

Taint flow to a dangerous sink

req.body.X reaches raw SQL, exec, fs path, or eval without going through a parser. ESLint-plugin-security greps for sinks; this detector follows the variable that lands in them.

high producer-consumer-field-drift

Server reads, no client sends

Handler destructures { email, role } from req.body, but every fetch / axios callsite for that URL ships only email. Silent undefined in production.

med schema-shape-divergence

Model and DTO drifted

Two exported types share a name stem (User vs UserDTO) and diverge by 1-2 keys — the classic "added a column, forgot the DTO" bug. tsc never compares unrelated types.

Full detector reference with severities + tunables: docs/DETECTORS.md.

cross-service races

The race that lives between services.

ESLint walks one file. SonarQube walks one repository. Both miss the race condition that spans two services. rothunter parses every linked package into one symbol graph and runs the race detectors cross-workspace — the only place the bug is visible.

how it works

Deterministic detectors. Local LLM verdicts.

1

Parse + graph

ts-morph builds a symbol + import graph across every linked workspace. Cross-package re-exports stitched via package.json#name.

2

Deterministic pass

33 detectors emit findings with stable fingerprints (23 ON by default — the rest cover ground ESLint already does). No flake — same code, same findings, every run.

3

LLM verdict

A local llama.cpp model triages each finding with project conventions (CLAUDE.md / AGENTS.md / …) in scope. Auto-routes obvious FPs.

project conventions are authoritative

rothunter reads CLAUDE.md, AGENTS.md, .cursorrules, copilot-instructions.md, CONTRIBUTING.md and feeds them to the LLM verdict. If your repo says "three similar lines beats premature abstraction", the verdict on a duplicate-function finding flips to FP — the project rule overrides generic best-practice. This is the SonarQube differentiator.
